# LeetCode 速查（leetcode-practice）

> 对应练习区：`leetcode-practice/`（每题 `src` 模板 + `tests` 验收 + `solutions` 参考）。
> 思路与复杂度速记，面试前过一遍即可。练习题分两档：下方 **中等** 为原题区（01–10），**简单** 为热身区（11–16）。

## 1. 无重复字符的最长子串（LC 3）
**滑动窗口**：`right` 右扩，遇到重复字符就把 `left` 一直右移到重复位置之后；用 `set` 记录窗口内字符。
- 时间 O(n)，空间 O(min(n, 字符集))。
- 易错：移除重复字符时用 `while s[right] in seen` 连续 `left++`，别只移一格。

## 2. 最长回文子串（LC 5）
**中心扩展**：每个位置当中心，分别扩 `(i,i)` 与 `(i,i+1)` 两种；记录最大长度对应的起点。
- 时间 O(n²)，空间 O(1)。
- 也可 Manacher 做到 O(n)，面试一般不要求。

## 3. 三数之和（LC 15）
**排序 + 双指针**：先排序，固定 `i`，`left=i+1, right=n-1` 夹逼；`i` 与 `left/right` 各自跳过重；`nums[i] > 0` 可提前 break。
- 时间 O(n²)，空间 O(1)（忽略排序）。
- 易错：去重要同时跳过 `i` 和相邻相等的 `left/right`，否则答案重复。

## 4. 电话号码的字母组合（LC 17）
**迭代/回溯**：用 `mapping` 字典，每读一位把已有前缀与当前位字母全组合；空串返回 `[]`。
- 时间 O(3^m·4^n)，空间同上。

## 5. 括号生成（LC 22）
**回溯**：维护 `open/close` 计数；`open < n` 才能加左括号，`close < open` 才能加右括号；长度到 `2n` 收口。
- 时间 O(4^n / √n)（卡特兰数），空间 O(n)。

## 6. 全排列（LC 46）
**回溯 + used 数组**：每层遍历未使用的元素，`path` 满 `n` 个即收口；用完回溯（pop + 置 False）。
- 时间 O(n·n!)，空间 O(n)。

## 7. 课程表（LC 207）
**拓扑排序 / 环检测**：建邻接表 + 入度表，BFS 从入度 0 出发；能访问完所有节点即无环（可修完）。
- 时间 O(V+E)，空间 O(V+E)。
- 等价 DFS 三色标记法。

## 8. 岛屿数量（LC 200）
**DFS/BFS 淹没**：遇 `'1'` 计数 +1，并把它连成一片的陆地全改成 `'0'`（避免重复计）。
- 时间 O(m·n)，空间 O(m·n)（递归栈/队列）。

## 9. 除自身以外数组的乘积（LC 238）
**前后缀分解**：`res[i] = 前缀积(0..i-1) × 后缀积(i+1..n-1)`；两遍扫描，不用除法。
- 时间 O(n)，空间 O(1)（输出数组不计入）。
- 易错：题目要求「不能用除法」且 O(n)，前缀后缀法是标准解。
- **JS 专属坑**：`0 * 负数` 会得到 `-0`，而 `assert.deepStrictEqual` 区分 `-0` 与 `0`，返回前用 `res.map((v) => (v === 0 ? 0 : v))` 归一化（Python 无此问题）。

## 10. 单词搜索（LC 79）
**回溯 + 标记**：从每个格子 DFS，匹配 `word[idx]`；访问过的位置临时置 `#` 防走回头，回溯还原。
- 时间 O(m·n·3^L)，空间 O(L)（递归深度）。

## 简单（Easy）热身区

## 11. 两数之和（LC 1）
**哈希表**：边走边把「值 → 下标」存进 `Map`；对当前数 `nums[i]`，看 `target - nums[i]` 是否已在表里，在即命中 `[seen, i]`。
- 时间 O(n)，空间 O(n)。
- 对比暴力 O(n²)；哈希把「再找一个数」降到 O(1)。
- 易错：先查表再入表，避免同一个元素用两次。

## 12. 有效的括号（LC 20）
**栈**：遇左括号入栈；遇右括号看栈顶是否匹配（出栈），不匹配即 false；最后栈须为空。
- 时间 O(n)，空间 O(n)。
- 易错：`([)]` 看似左右数量对，但顺序错——栈能识别；空串算合法。

## 13. 爬楼梯（LC 70）
**动态规划（滚动数组）**：`dp[i] = dp[i-1] + dp[i-2]`；只保留前两项，用 `a, b` 滚动。`dp[1]=1, dp[2]=2`。
- 时间 O(n)，空间 O(1)。
- 本质是第 n 个斐波那契数。

## 14. 最大子数组和（LC 53 · Kadane）
**贪心 / 动态规划**：`cur = max(nums[i], cur + nums[i])`（接上前面 or 从当前重新开始）；`best` 跟踪全局最大。
- 时间 O(n)，空间 O(1)。
- 易错：全负数时返回最大那个负数，不能初始化成 0；`best` 初值取 `nums[0]`。

## 15. 存在重复元素（LC 217）
**哈希集合**：边走边 `Set`，命中即 true；走完无重复 false。
- 时间 O(n)，空间 O(n)。
- 进阶：先排序再比相邻也可 O(n log n)。

## 16. 有效的字母异位词（LC 242）
**计数 / 哈希表**：长度不同直接 false；用 `Map` 统计 `s` 频次，再对 `t` 逐个减，出现负值或最后有剩即 false。
- 时间 O(n)，空间 O(1)（字符集固定）。
- 也可排序两串后比较，但计数法更稳。

---

**一句话速记**：窗口/双指针解子串与求和，回溯解排列组合与搜索，拓扑解依赖，网格 DFS 解连通块，前后缀解乘积；哈希表是「两数之和 / 异位词 / 去重」的万能钥匙，栈管括号，Kadane 管最大子段，滚动数组管斐波那契式递推。

## 附录：数据结构方法速查

刷题常混的"增 / 查 / 删 / 长度"，对照记：

| 结构 | 增 | 查「是否存在」 | 取 / 删 | 遍历 | 长度 |
|------|----|------|------|------|------|
| **Map** | `set(k, v)` | `has(k)` | `get(k)` / `delete(k)` | `forEach` / `keys`·`values` | `.size` |
| **Set** | `add(v)` | `has(v)` | `delete(v)` | `forEach` / `keys`(=values) | `.size` |
| **Array** | `push` / `unshift` | `includes(v)` ⚠️ | `pop` / `shift` / `splice` | `forEach` / `map`·`filter`·`reduce` | `.length` |

易错点：
- **「在不在里面」三兄弟**：`map.has(key)` / `set.has(value)` / `arr.includes(value)`。数组别用 `indexOf` 当 `has` 用——`indexOf` 返回下标（找不到是 `-1`），`includes` 才返回布尔。
- **长度**：Map/Set 用 `.size`，Array 用 `.length`。
- **删**：Map/Set 用 `delete(k/v)`；Array 没有 `delete` 方法（用 `pop`/`shift`/`splice`），且**不要写 `delete arr[2]`**——那只把元素变 `empty`、长度不变，是坑。
- **铁律**：Map/Set 是特殊对象，必须用专属方法（`set`/`get`/`has`/`add`），不能用 `obj[key] = val` 中括号赋值（那只是挂了个无关属性，数据没进容器）。
